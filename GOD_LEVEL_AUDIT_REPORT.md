## **II. CRITICAL BLOCKERS (P0 - FIX IMMEDIATELY)**

These issues must be resolved before any other work.

### **1. (BUG) Dasha Calculation is Corrupted**

*   **File:** `backend/src/lib/btr/dasha-builder.ts` (Line 146)
*   **Problem:** The `createDashaEntry` function incorrectly assigns parameters, swapping `maha` and `antar` values. This corrupts every Dasha sequence generated by the system.
*   **Impact:** **Catastrophic.** The AI is making decisions based on fundamentally incorrect astrological data. All timing-related analysis is wrong.
*   **Required Fix:** Correct the parameter assignment in the return statement.
    ```typescript
    // FROM (BROKEN)
    return { maha, antar: maha, pratyantar: antar, sukshma, prana, startEnd };
    
    // TO (CORRECT)
    return { maha, antar, pratyantar, sukshma, prana, startEnd };
    ```

### **2. (Security) Cross-Site Scripting (XSS) Vulnerability**

*   **Files:** `UnifiedAIPanel.tsx`, `use-stream-progress.ts`
*   **Problem:** AI-generated content is rendered directly to the DOM without proper sanitization. The `cleanReasoningText` function is insufficient and does not prevent malicious HTML or script injection.
*   **Impact:** A malicious actor could craft AI output that steals user data, takes over accounts, or performs unauthorized actions.
*   **Required Fix:**
    1.  Implement a robust, industry-standard HTML sanitization library (like `DOMPurify`).
    2.  Apply sanitization to all content received from the AI before rendering it.
    3.  Implement a strict Content Security Policy (CSP).

### **3. (Stability) Race Conditions in UI**

*   **File:** `lib/use-stream-progress.ts`
*   **Problem:** The hook that manages streaming data has multiple race conditions. The SSE connection and polling mechanism can update state concurrently, and stale closures lead to inconsistent UI.
*   **Impact:** The UI shows incorrect progress, stale data, and can easily get into a broken state.
*   **Required Fix:**
    1.  Refactor `use-stream-progress.ts` to use a state machine or a more robust state management library.
    2.  Ensure all asynchronous operations are properly sequenced and cancellable.
    3.  Remove redundant `fetchProgress` calls.

### **4. (Stability) Memory Leaks & Unhandled Errors**

*   **Files:** `app/rectify/[id]/page.tsx`, `AnalysisPipelineTracker.tsx`, `UnifiedAIPanel.tsx`
*   **Problem:**
    *   No top-level error boundaries. A single component crash takes down the entire page.
    *   `setInterval` and `requestAnimationFrame` calls are not consistently cleaned up, leading to memory leaks and performance degradation over time.
*   **Impact:** The application is unstable, prone to crashing, and will become slower the longer it is used.
*   **Required Fix:**
    1.  Wrap the main analysis page in a global `AnalysisErrorBoundary` component.
    2.  Add granular error boundaries to individual UI sections.
    3.  Ensure every `useEffect` hook that sets up a timer or subscription has a proper cleanup function that clears it.

## **III. HIGH-PRIORITY ISSUES (P1 - FIX BEFORE PRODUCTION)**

### **1. (Logic) Insufficient Astrological Data in Stage 2**

*   **File:** `backend/src/lib/btr/stages/stage2-batch-tournament.ts`
*   **Problem:** The initial "Batch Tournament" stage is using a Dasha depth of 2 (Maha-Antar) instead of the required depth of 3 (Pratyantar). This is not enough detail to accurately correlate life events.
*   **Impact:** The system may be prematurely eliminating valid birth time candidates.
*   **Required Fix:** Change `dashaDepth` from `2` to `3` for Stage 2.

### **2. (Logic) Missing D60 Deity Data**

*   **File:** `backend/src/lib/btr/data-package-builder.ts`
*   **Problem:** The D60 (Shashtyamsa) divisional chart is being calculated, but the associated deities are not being populated. The AI prompts reference these deities, but they are receiving empty data.
*   **Impact:** A key "God-Tier" analysis feature is completely non-functional.
*   **Required Fix:** Implement the logic to correctly calculate and populate the D60 deities for each planet.

### **3. (Accessibility) WCAG 2.1 AA Compliance Failure**

*   **Files:** All analysis page components.
*   **Problem:** The application is completely inaccessible to users with disabilities. There are no ARIA attributes, no keyboard navigation, and no focus management.
*   **Impact:** You are excluding a segment of your user base and may be in violation of legal accessibility requirements.
*   **Required Fix:** Conduct a full accessibility audit and implement all necessary ARIA roles, states, and properties. Ensure all interactive elements are reachable and operable via the keyboard.
